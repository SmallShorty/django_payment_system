
## Описание проекта
Этот проект представляет собой Django-приложение с интеграцией платежной системы Stripe.

### Стек технологий
- Backend: Python 3.11, Django
- Database: SQLite (используется для демонстрации и простоты развертывания)
- Containerization: Docker, Docker Compose
- Integrations: Stripe SDK, Exchange Rate API

## Инструкция по запуску

### 1. Клонирование репозитория
    git clone https://github.com/SmallShorty/django_payment_system.git
    cd django_payment_system


### 2. Настройка переменных окружения (.env)

В репозитории отсутствует файл `.env`, так как он содержит секретные ключи. Чтобы проект заработал, создайте его на основе примера:

1. Найдите в корне файл `.env.example`.
2. Скопируйте его и назовите просто `.env`:

3. Откройте `.env` и вставьте ваши реальные ключи для **Stripe** и **API курсов валют**.

### 3. Сборка и запуск Docker-контейнера

Убедитесь, что у вас установлен Docker. Выполните команду для сборки образа и запуска приложения:

```bash
docker-compose up --build

```

*После завершения сборки приложение будет доступно по адресу: **[http://127.0.0.1:8000*](https://www.google.com/search?q=http://127.0.0.1:8000)**

### 4. База данных

В данном проекте в качестве базы данных используется **SQLite**. Это сделано в демонстрационных целях, чтобы вам не нужно было настраивать тяжелые серверы БД (вроде PostgreSQL или MySQL) при первом знакомстве с проектом.

Файл базы данных `db.sqlite3` создается внутри контейнера и автоматически синхронизируется с вашей локальной папкой. Чтобы подготовить структуру таблиц, выполните миграции:

```bash
docker-compose exec web python manage.py migrate

```

### 5. Создание суперпользователя

Для доступа в панель администратора Django (`/admin`) вам необходимо создать аккаунт администратора. Выполните команду и следуйте инструкциям в терминале:

```bash
docker-compose exec web python manage.py createsuperuser

```
